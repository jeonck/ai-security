# Project-S Rules & Customizations

이 문서에는 프로젝트 "Project-S"의 문서 작성 및 시스템 배포 가이드라인과 룰셋을 정의합니다. 모든 AI 에이전트는 본 가이드라인을 엄격히 준수하여 코드를 변경하거나 문서를 작성해야 합니다.

---

## ✍️ 한글 마크다운 강조(Bold) 처리 규칙

### 1. 문제의 정의
Hugo의 기본 마크다운 파서인 **Goldmark**는 CommonMark 규격을 엄격히 따릅니다. 이 규격에서는 강조 기호(`**` 또는 `*`)가 열리고 닫히는 규칙(Left/Right-Flanking Delimiter)을 문자 경계에 따라 엄격히 판단합니다.
이로 인해 한글 마크다운 작성 시 다음과 같은 구조에서 **볼드체 렌더링이 깨지고 생텍스트(`**`)가 그대로 노출되는 치명적인 오류**가 발생합니다.

* **오류 케이스 1**: 강조 기호 바로 뒤에 공백 없이 한글 조사(가, 는, 을, 를, 의, 에 등)가 올 때
  * 예: `**거버넌스**가` (Goldmark 파서 오작동으로 볼드 미적용)
* **오류 케이스 2**: 강조 기호 내부에 영문 괄호 `()`, 쉼표 `,` 또는 인라인 코드블록이 섞여 있고, 닫는 기호 직전에 괄호가 올 때
  * 예: `**보안 게이트웨이(Reverse Proxy)**에` (괄호와 조사의 결합으로 볼드 적용 완전히 실패)

### 2. 해결 방안 및 표준 작성 규격
이 문제를 원천 차단하고 의미론적 강조 범위와 한글 조사 맞춤법(붙여쓰기)을 모두 준수하기 위해 **HTML `<strong>` 태그 사용을 의무화**합니다. (이 사이트의 `hugo.toml`에는 `unsafe = true`가 적용되어 있어 HTML 태그 렌더링을 완전히 지원합니다.)

#### [규칙 A] 영문 괄호나 특수문자가 섞인 강조 ➔ `<strong>` 사용
강조할 텍스트에 영문 괄호, 쉼표, 하이픈 등 특수 기호가 포함되는 경우 마크다운 `**` 대신 HTML `<strong>` 태그를 사용하고, 한글 조사는 태그 바깥에 붙여 씁니다.
* ❌ **나쁜 예**: `**보안 게이트웨이(리버스 프록시, Reverse Proxy)**에` (렌더링 실패)
* ❌ **나쁜 예**: `**보안 게이트웨이(리버스 프록시, Reverse Proxy)에**` (조사까지 강조되어 의미론적으로 어색함)
* ⭕ **올바른 예**: `<strong>보안 게이트웨이(리버스 프록시, Reverse Proxy)</strong>에` (완벽하게 렌더링 성공)

#### [규칙 B] 단순 한글 강조 ➔ 띄어쓰기 또는 HTML 태그 사용
단순 한글 텍스트를 강조할 때 조사가 바로 붙는다면, 문맥에 따라 띄어 쓰거나 `<strong>`을 활용합니다.
* ❌ **나쁜 예**: `이것은 **중요한 문제**입니다.`
* ⭕ **올바른 예**: `이것은 <strong>중요한 문제</strong>입니다.` (조사와 붙여 쓸 때)
* ⭕ **올바른 예**: `이것은 **중요한** 문제입니다.` (단어 단위로 쪼개어 띄어 쓸 수 있을 때)
